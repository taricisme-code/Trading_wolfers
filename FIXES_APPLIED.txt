✅ ISSUE 2: Not Responsive on Full Screen
-------------------------------------------
Problem:
- All Scene objects created with fixed size: new Scene(root, 1200, 700)
- Window couldn't expand beyond 1200x700
- Layout didn't adapt to full screen
- Elements stayed small and cramped when maximized

Solution:
- Changed ALL Scene creations from: new Scene(root, 1200, 700)
                             to: new Scene(root)
- Scenes now inherit stage dimensions automatically
- Added stage.setMaximized(true) to MainApp.java
- Window starts maximized on launch
- Layout components now grow to fill available space

Files Modified:
1. MainApp.java
   - Added: stage.setMaximized(true);
   - Keeps minWidth/minHeight for minimum usable size

2. DashboardController.java
   - Changed loadView() to create Scene without fixed dimensions
   - Scene automatically fills stage

3. TradingController.java
   - Changed loadView() to create responsive Scene
   - Trading terminal now fills full screen

4. LoginController.java (2 locations)
   - handleRegisterNavigation(): Scene without fixed size
   - loadDashboard(): Scene without fixed size

5. RegisterController.java (2 locations)
   - handleLoginNavigation(): Scene without fixed size
   - loadDashboard(): Scene without fixed size


✅ HOW THE FIX WORKS
---------------------
Before:
  Scene scene = new Scene(root, 1200, 700);  // Fixed size!
  
After:
  Scene scene = new Scene(root);  // Inherits from stage dimensions

JavaFX Behavior:
- Scene(root, width, height) = Fixed size, won't grow beyond specified dimensions
- Scene(root) = Adapts to stage size, grows/shrinks with window

The BorderPane, SplitPane, VBox, and HBox layouts in the FXML files already
had proper grow priorities and constraints. They were just constrained by the
fixed Scene dimensions. Now they can expand freely!

✅ TECHNICAL DETAILS
---------------------
Scene Creation Best Practice:

Bad (Fixed):
  Scene scene = new Scene(root, 1200, 700);
  - Hard-coded dimensions
  - Won't grow beyond this size
  - Ignores stage size
  
Good (Responsive):
  Scene scene = new Scene(root);
  - Adapts to stage dimensions
  - Grows with window
  - Respects layout constraints

Stage Configuration:
  stage.setMinWidth(1000);    // Minimum usable width
  stage.setMinHeight(600);    // Minimum usable height
  stage.setMaximized(true);   // Start full screen
  
This ensures:
- Window can't be too small to be usable
- Window starts at maximum comfortable size
- User can resize freely within these bounds
